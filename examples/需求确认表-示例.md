# 需求确认表 — 填写示例

> 以下为脱敏后的真实案例，展示一份完整的需求确认表长什么样。

---

## 项目背景

某互联网公司运营团队想了解用户对"游戏权益"的真实需求，决定Q3是否新增游戏相关内容。

---

## 填写结果

| 字段 | 填写内容 |
|------|---------|
| **项目名称** | 游戏权益用户需求调研 |
| **业务方 / 需求提出人** | 嘉欣（运营岗） |
| **研究核心问题** | 用户对游戏权益的真实需求是什么？现有权益的使用率和满意度如何？ |
| **支撑的决策动作** | 决定Q3是否新增《原神》定向流量包，以及现有25款游戏权益是否需要调整 |
| **产出物形态** | ☑ 结论页PPT |
| **报告受众** | ☑ 内部管理层 |
| **数据回收截止** | 7月15日 |
| **报告提交节点** | 7月22日 |
| **备注** | 嘉欣希望同时了解用户对"无对赌合作模式"的接受度，作为附加模块 |

---

## 方法选择

| 研究问题 | 推荐方法 | 理由 |
|---------|---------|------|
| 用户对游戏权益的真实需求 | 定量为主（问卷） | 需要量化各权益的偏好排序，样本量大 |
| 现有权益的使用体验 | 定性补充（访谈） | 需要了解使用场景和痛点，问卷问不出来 |
| 竞品游戏权益布局 | 桌面研究 | 纯二手数据，不需要用户参与 |

**最终方案**：问卷（N=600）+ 深度访谈（N=21）+ 竞品桌面研究
