# 完整案例 Walkthrough

> 这是一份**从头到尾的故事线**，带你走一遍用研SOP的完整流程。
> 本案例基于作者标注的脱敏案例改写，本次修订未独立核验其真实性。
>
> 读完这篇，你就知道怎么用这套SOP了。
>
> **数据与证据声明**
> - 作者称公司名称、品牌名称和业务数据已经脱敏，本次修订未接触原始材料，未独立核验。
> - 案例中的问卷样本量（600份）为【规划假设】，应根据估计或比较目标、误差、效应量和功效等设计。
> - 访谈样本量（20场）为【规划假设】，应根据用户异质性、关键分群覆盖和信息饱和度设计。
> - 竞品数量（3-5家）为【规划假设】，应根据决策范围、候选池覆盖和新增信息价值决定。
> - 回收进度表中的数字为【虚构流程示意】。
> - 执行时长（5分钟、2分钟、30分钟、7-10天、3-5天、2-3天、1天、约3周）为【虚构流程示意】。
> - 流程根据案例方法回顾整理，不属于研究证据。

---

## 背景故事

你是某公司的用户研究员。运营同事嘉欣找到你说：

> "领导想看看游戏权益这块用户到底要什么，你帮忙做个调研呗。"

你的第一反应可能是：游戏权益？什么游戏？什么用户？做问卷还是访谈？什么时候要？

**别急，先填表。**

---

## 第一步：需求确认（5分钟）【虚构流程示意】

打开 `templates/需求确认表.md`，填这个表：

| 字段 | 你填的内容 |
|------|-----------|
| 项目名称 | 游戏权益用户需求调研 |
| 业务方 / 需求提出人 | 嘉欣（运营） |
| 研究核心问题 | 用户对游戏权益的真实需求是什么？现有权益的使用率和满意度如何？ |
| 支撑的决策动作 | 决定Q3是否新增《原神》定向流量包，以及25款游戏权益是否需要调整 |
| 产出物形态 | ☑ 结论页PPT |
| 报告受众 | ☑ 内部管理层 |
| 数据回收截止 | 7月15日 |
| 报告提交节点 | 7月22日 |

**填完之后你要做一件事：** 把这张表发给嘉欣，确认她同意。如果她说"我也不太确定，领导就是说看看"——**退回，让她问清楚领导要什么**。

> 💡 **为什么要先填表？** 需求和决策没有对齐，会导致研究结果无法支持业务行动，这是常见的项目失败原因。你做了一堆东西，领导说"这不是我要的"。

---

## 第二步：选方法（2分钟）【虚构流程示意】

填完表后，你看一下这个问题类型：

- 你不知道用户要什么 → **探索性**
- 你还要验证某个假设（原神流量包值不值得做）→ **验证性**

所以你的方案是：**问卷（大量用户）+ 深度访谈（深入挖掘）+ 竞品研究（看看对手在做什么）**。

| 方法 | 样本量 | 目的 |
|------|--------|------|
| 问卷 | 600份 | 量化各权益的偏好排序 |
| 深度访谈 | 20场 | 挖掘使用场景和痛点 |
| 竞品研究 | 3-5家 | 看看电信移动在做什么 |

> 【规划假设】问卷样本量应根据估计或比较目标、误差、效应量和功效等设计；访谈样本量应根据用户异质性、关键分群覆盖和信息饱和度设计；竞品数量应根据决策范围和新增信息价值决定。以上数字为本案例中的规划假设，非通用标准。

---

## 第三步：设计问卷（30分钟）【虚构流程示意】

打开 `prompt-templates/02-研究设计.md`，复制"2.1 问卷设计"的提示词，替换 `{}` 中的内容，发给AI：

```
请帮我设计一份问卷：

研究目的：了解某互联网产品用户对游戏权益的真实需求，决定Q3是否新增《原神》定向流量包
目标人群：某互联网产品现有用户，18-30岁，有游戏行为
分群方式：按游戏类型分——腾讯系玩家/非腾讯系玩家/不玩游戏
预计样本量：600份
问卷时长：控制在5分钟以内
...
```

AI会给你一份问卷初稿。**但先别用——让另一个AI审查一遍。**

打开另一个 AI 对话（如另一个模型或新开对话），把初稿发给它，用"2.2 问卷审查"的提示词：

```
请审查以下问卷，找出问题：
{粘贴问卷}

审查标准：
1. 题目措辞是否有歧义
2. 选项覆盖度是否完整
3. 跳题逻辑是否有漏洞
...
```

审查模型可能会告诉你："这道题选项漏了'其他'"——**这就是Agent Loop在起作用**。

你根据审查意见改完，问卷就定稿了。

> **独立上下文审查说明**：独立上下文或审查模型可以帮助发现逻辑漏洞、遗漏和反例，但不能提高事实证据强度。只有新增原始数据、外部来源、重新计算或人工核查才能提高证据强度。

---

## 第四步：投放+访谈（7-10天）【虚构流程示意】

问卷定稿后投放，同时开始访谈。

**每天看一眼回收进度：** 【虚构流程示意——以下数字均为虚构】

| 日期 | 总回收 | 腾讯系 | 非腾讯系 | 不玩游戏 | 状态 |
|------|--------|--------|---------|---------|------|
| 7/3 | 127 | 52 | 38 | 37 | 正常 |
| 7/5 | 289 | 118 | 89 | 82 | 正常 |
| 7/8 | 467 | 191 | 143 | 133 | ⚠️ 非腾讯系偏少 |
| 7/10 | 601 | 243 | 179 | 179 | ✅ 达标 |

如果发现某群欠采，赶紧补投。

**访谈那边，每场用 `templates/访谈记录表.md` 记录：**

```
受访者编号：P07
访谈日期/时长：2026-07-05 / 28分钟
受访者类型：☑ 已激活 · 腾讯系游戏玩家
```

**核心发现记三栏：**

| 原话引用 | 行为归因 | 研究者判断 |
|---------|---------|-----------|
| "定向流量听起来挺好，但我根本不知道哪些算定向" | 用户对权益的透明度认知模糊 | ⭐ 权益价值感知低的主因是信息不透明 |
| "如果能送原神的月卡，我肯定天天看公众号" | 用户对"内容型权益"感知价值更高 | ⭐ 内容型权益 > 基础设施型权益 |

---

## 第五步：分析数据（3-5天）【虚构流程示意】

### 定量：问卷数据分析

用 `prompt-templates/04-数据分析.md` 里的提示词：

```
帮我清洗这份问卷数据：
- 剔除基于完成时长分布的异常样本（阈值根据预试数据确定）
- 剔除关键题全空的样本
- 剔除逻辑矛盾的样本

然后做交叉分析：按"游戏类型"分群，对比各群对游戏权益的偏好排序。
```

### 定性：访谈编码

同样用提示词：

```
帮我做主题编码，以下是访谈的关键发现：
{粘贴所有访谈的核心发现}

要求：
1. 逐条提取核心行为动机
2. 归并共性主题，标注出现频次
3. 找反例
4. 分级：核心发现/次级发现/观察备注
```

### 交叉验证

分析完后，**用Agent Loop审查结论**：

```
请审查我的分析结论：

1. 用户对游戏权益的核心需求是"信息透明度"而非"权益数量"
2. 至少30%的用户游戏偏好已跨出腾讯系
3. 内容型权益感知价值 > 基础设施型权益
4. ...

审查标准：每条结论是否有数据支撑？是否过度推断？有没有反例？
```

---

## 第六步：写报告（2-3天）【虚构流程示意】

### 先搭结构

```
请帮我搭一份研究报告的结构：

核心发现（3条）：
1. 权益透明度 > 权益丰富度
2. 游戏偏好已跨出腾讯系
3. 内容型权益感知价值更高

受众：内部管理层
形态：结论页PPT
```

### 结论先行改写

你写完初稿后，用提示词让它改格式：

```
请帮我把这段改成"结论先行"格式：
{你写的段落}

改写规则：
1. 第一句话必须是结论
2. 数据跟在结论后面
3. 删除描述性堆砌
```

### 终审

报告写完后，**用Agent Loop做最后检查**：

```
请终审这份报告：
1. 主结论是否在第一页可见
2. 每条建议是否可执行
3. 关键数字是否标注来源
4. 是否可以直接交给领导
```

---

## 第七步：交付+复盘（1天）【虚构流程示意】

交完报告后别忘了：

1. **归档**：问卷原件、访谈记录、清洗后数据、最终报告存好
2. **复盘**：哪些方法好用？哪些坑踩了？下次怎么改？
3. **沉淀**：把问卷模板、访谈提纲存到复用库，下次同类项目直接用

---

## 完整流程图

```
嘉欣说"做个调研"
    ↓
[5min] 填需求确认表 → 发给嘉欣确认 【虚构流程示意】
    ↓
[2min] 选方法：问卷+访谈+竞品 【虚构流程示意】
    ↓
[30min] AI出问卷初稿 → 另一个AI审查 → 改完定稿 【虚构流程示意】
    ↓
[10天] 投放问卷 + 执行访谈（并行） 【虚构流程示意】
    ↓
[5天] 清洗数据 → 交叉分析 → 访谈编码 【虚构流程示意】
    ↓
[审查] Agent Loop审查结论（另一个AI找问题）
    ↓
[3天] 搭报告结构 → 写内容 → 结论先行改写 【虚构流程示意】
    ↓
[审查] Agent Loop终审报告
    ↓
[1天] 交付 → 归档 → 复盘 【虚构流程示意】
```

**总计：约3周**【虚构流程示意】（含投放等待时间）
