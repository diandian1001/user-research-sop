# AI 用户研究操作系统 V4.0

**AI User Research Operating System — 可移植、可执行、可追踪的研究工作包**

[English](README.md)

[![License: MIT](https://img.shields.io/badge/License-MIT-7c5cfc.svg)](https://opensource.org/licenses/MIT)
[![Stars](https://img.shields.io/github/stars/diandian1001/user-research-sop?style=social)](https://github.com/diandian1001/user-research-sop)
[![Version](https://img.shields.io/badge/Version-V4.0-10b981.svg)](https://github.com/diandian1001/user-research-sop)

把这个仓库交给 AI 或 Agent，它可以从业务决策定义开始，自动选择研究方法、生成材料、维护项目状态、管理证据，并输出可直接使用的研究交付物。

本仓库提供三种入口：

| 入口 | 文件 | 适用场景 |
|---|---|---|
| Agent 仓库模式 | `AGENTS.md` | Hermes、Codex、Claude Code 等能读取仓库的 Agent |
| Skill 模式 | `SKILL.md` | 支持系统指令、Project Knowledge 或 Skill 的平台 |
| 任意 AI 便携模式 | `PORTABLE.md` | 无法完整读取 GitHub 仓库的普通 AI 对话 |

> 清晰任务默认使用“快速执行模式”。AI 只在缺失信息会实质改变方法或结论时提问。

## 一句话启动

### 仓库模式

```text
读取 AGENTS.md，并使用这个仓库完成以下用户研究任务：
[粘贴任务]
```

### Skill 模式

```text
使用 SKILL.md，以快速执行模式完成：
[粘贴任务]
```

### 任意 AI 模式

复制 `PORTABLE.md` 全文，然后追加任务。

AI 的首轮输出应包含：

* 任务理解；
* 业务决策；
* 执行模式；
* 推荐方法；
* 已确认信息；
* 默认假设；
* 关键缺口；
* 下一步动作。

## 30 秒看懂

**输入** — 告诉 AI：
> *"帮我分析为什么新用户第一周后就不回来了"*

**过程** — AI 会：
1. 问几个澄清问题（定义"不回来"、时间线、给谁看）
2. 推荐最合适的方法（大概率是漏斗分析 + 访谈）
3. 生成数据提需模板和访谈提纲
4. 一步步指导你完成分析
5. 帮你写报告

**输出** — 你会得到：
- 一份结构化的研究方案
- 可直接使用的数据提需和访谈材料
- 一套分步骤的分析框架
- 一份包含优先级行动建议的完整报告

→ [完整案例 Walkthrough](examples/完整案例-Walkthrough.md) | [真实脱敏案例](examples/case-telecom-conversion-diagnosis.md)

## 支持什么

### 当前支持的研究任务

| 研究任务 | 方法 | 最适合 |
|---------|------|--------|
| 理解用户动机 | 深度访谈提纲生成 | 探索性研究——问"为什么" |
| 大规模验证假设 | 问卷调研设计 | 量化验证——问"是什么"和"有多少" |
| 竞品研究 | 桌面研究 | 了解别人在做什么 |
| 转化与留存分析 | 漏斗分析 | 找到用户在哪个环节流失 |
| 可用性测试 | 任务式观察协议 | 用户能否完成关键任务 |
| 概念或原型测试 | 价值主张与概念评估 | 用户是否理解和认同某个方案 |

### 当前不支持（需要专门工具或完整执行环境）

- 眼动/生理测量研究
- A/B 测试设计与完整统计功效分析
- 长期追踪/日记研究
- 田野民族志
- 无障碍审计

如果你的研究需求超出当前范围，AI 会诚实告诉你，而不是假装能覆盖。

### 适合谁

- 需要做用户研究但缺乏正式训练的产品经理
- 想用用户证据验证设计决策的设计师
- 寻找结构化工作流的初级研究者
- 想标准化研究流程的团队

### 不适合谁

- 需要严格方法论论证的学术研究者
- 受监管的临床/医学研究
- 研究成果将作为法律证据的场景
- 完全没听过"用户研究"的纯新手——需要对研究概念有基本了解

## 怎么用

### ChatGPT

1. 打开新对话
2. 粘贴 `https://github.com/diandian1001/user-research-sop` 并说 *"使用这个仓库里的 SKILL.md"*
3. 如果 ChatGPT 无法直接读取 URL，复制 `SKILL.md` 全文，作为第一条消息粘贴
4. 然后描述你的研究需求

### Claude

1. 在 Claude 中创建新 Project
2. 将 `SKILL.md` 添加为 Project Knowledge
3. 开始对话，描述你的研究需求
4. Claude 会自动按照引导脚本运行

### Hermes

1. 将仓库地址或 `SKILL.md` 添加到 Hermes 会话
2. 说：*"帮我做用户研究"*
3. Hermes 会在有搜索、文件分析等工具能力时自动调用

## AI 会引导你做什么

| 步骤 | 发生什么 |
|------|---------|
| Step 1 理解需求 | 4 个问题（选择题+填空），搞清楚你要研究什么 |
| Step 2 推荐方法 | 决策树根据你的问题和时间线推荐最合适的方法 |
| Step 3 生成材料 | AI 按规则生成访谈提纲或问卷初稿，不套固定模板 |
| Step 4 分析指导 | 定量 5 步 / 定性 4 步，一步步引导你完成分析 |
| Step 5 报告生成 | 填空式模板，自动生成完整 Markdown 报告 |

## 重要局限

使用前请阅读：

1. **AI 不能代替研究伦理判断。** 知情同意、隐私保护、数据伦理处理是你的责任。
2. **AI 不能保证样本质量。** 它可以建议样本量，但不能验证你的受访者是否具有代表性。
3. **AI 输出需要人工验证。** 事实、数字和推论在使用前必须核对。AI 可能误分类、误解或遗漏细节。
4. **涉及敏感数据需要本地化或脱敏处理。** 不要将包含个人身份信息的原始用户数据粘贴到公共 AI 服务中。
5. **本项目不覆盖全部用户研究方法。** 当前支持范围见上方"支持什么"。
6. **不得使用本项目伪造调研结果或用户证据。** 这是一个引导工具，不是真实研究的替代品。

## V4.0 核心升级

- **三种执行模式**：快速执行、标准协作、引导教学；
- **Agent 加载协议**：通过 `AGENTS.md` 规定读取和执行顺序；
- **单文件便携版**：`PORTABLE.md` 可直接复制给任意 AI；
- **决策优先**：研究开始前明确业务决策；
- **项目状态管理**：跨阶段记录范围、口径、假设和进度；
- **Claim 级证据账本**：结论可以追溯到数据、原话和来源；
- **自动交付**：AI 根据证据自动生成报告，不要求用户重新填空；
- **方法规则修正**：取消固定 500 份样本、固定 5% 差异阈值和默认 NPS；
- **跨工作流路由**：复杂竞品分析转交专门竞品分析工作流；
- **Benchmark**：建立跨平台、重复运行的稳定性测试方案。
