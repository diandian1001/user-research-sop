# 阶段一：需求承接 — 提示词模板

> 直接复制粘贴到AI对话框中使用。`{}` 中的内容替换为你的实际信息。

---

## 1.1 需求确认（发给AI帮你梳理）

```
我接到了一个用研需求，请帮我梳理：

需求背景：{需求方说的内容，可以是聊天记录、邮件、口头描述}
需求方：{谁提的，什么职位}
期望产出：{报告/PPT/结论页/其他}
期望时间：{什么时候要}

请帮我：
1. 提炼出明确的研究目的（支撑什么决策）
2. 判断这个目的是否足够清晰，如果模糊，列出我需要追问的3个问题
3. 根据目的推荐研究方法（定性/定量/混合）
4. 倒排出关键里程碑（数据回收→分析→报告→交付）
```

**示例（替换后）：**
```
我接到了一个用研需求，请帮我梳理：

需求背景：领导说"想看看游戏权益这块用户到底要什么"
需求方：嘉欣，运营岗
期望产出：结论页PPT
期望时间：7月22日前

请帮我：
1. 提炼出明确的研究目的（支撑什么决策）
2. 判断这个目的是否足够清晰，如果模糊，列出我需要追问的3个问题
3. 根据目的推荐研究方法（定性/定量/混合）
4. 倒排出关键里程碑（数据回收→分析→报告→交付）
```

## 1.2 方法选择确认（发给AI帮你验证）

```
我准备用以下方法做研究，请帮我检查是否合理：

研究问题：{你要解决的问题}
拟用方法：{你选的方法}
数据来源：{问卷/访谈/行为数据/桌面研究}

请判断：
1. 这个方法和研究问题是否匹配
2. 有没有更好的替代方法
3. 这个方法的主要风险是什么
4. 如果我坚持用这个方法，需要额外注意什么
```

## 1.3 需求方对齐话术（发给AI帮你写邮件/消息）

```
请帮我写一段发给需求方的消息，确认以下内容：

需求方：{名字/职位}
需要确认的事项：{不确定的点}
语气：{正式/轻松}

要求：
- 不要质问，用确认的语气
- 把不确定的点包装成选择题
- 控制在100字以内
```

**示例：**
```
请帮我写一段发给需求方的消息，确认以下内容：

需求方：嘉欣，运营
需要确认的事项：研究目的是决定Q3游戏权益方向，还是评估现有权益效果
语气：轻松，同事关系

要求：
- 不要质问，用确认的语气
- 把不确定的点包装成选择题
- 控制在100字以内
```
