# 阶段二：研究设计 — 提示词模板

---

## 2.1 问卷设计（发给AI帮你出初稿）

```
请帮我设计一份问卷，信息如下：

研究目的：{你要研究什么}
目标人群：{谁来填，什么特征}
分群方式：{怎么分群，比如已购/未购、活跃/流失}
预计样本量：{多少份}
问卷时长：{希望控制在几分钟以内}

要求：
1. 首题必须是身份确认/筛选题
2. 通用题占比≤30%，其余按分群跳转
3. 题型优先用：场景描述题 > 强迫单选题 > 量表题
4. 强迫选择题不要出现"无偏好/都可以"选项
5. 每道题标注：属于通用模块还是哪个分群模块
6. 最后附一个跳题逻辑说明

请先出问卷结构（题目清单+分群标注），我确认后再出完整版。
```

**示例：**
```
请帮我设计一份问卷，信息如下：

研究目的：了解王卡用户对游戏权益的真实需求，决定Q3是否新增《原神》定向流量包
目标人群：王卡现有用户，18-30岁，有游戏行为
分群方式：按游戏类型分——腾讯系玩家/非腾讯系玩家/不玩游戏
预计样本量：600份
问卷时长：控制在5分钟以内

要求：
1. 首题必须是身份确认/筛选题
2. 通用题占比≤30%，其余按分群跳转
3. 题型优先用：场景描述题 > 强迫单选题 > 量表题
4. 强迫选择题不要出现"无偏好/都可以"选项
5. 每道题标注：属于通用模块还是哪个分群模块
6. 最后附一个跳题逻辑说明

请先出问卷结构（题目清单+分群标注），我确认后再出完整版。
```

## 2.2 问卷审查（发给审查模型找问题）

```
请审查以下问卷，找出问题：

{粘贴问卷全文}

审查标准：
1. 题目措辞是否有歧义（一道题只能理解成一种意思）
2. 选项覆盖度是否完整（有没有漏掉重要选项）
3. 跳题逻辑是否有漏洞（会不会有人跳过不该跳的题）
4. 有没有"引导性提问"（暗示用户选某个答案）
5. 强迫选择题是否真的有区分度
6. 问卷总时长是否合理

请逐题审查，每题给出：✅无问题 / ⚠️建议修改 / ❌必须修改
对于⚠️和❌，给出具体修改建议。
```

## 2.3 访谈提纲设计（发给AI帮你出初稿）

```
请帮我设计一份访谈提纲：

访谈对象：{什么用户，什么状态，比如已购/流失}
访谈时长：{计划多少分钟}
核心议题：{要探索的3-5个问题}
背景信息：{已知的背景，避免重复问}

要求：
1. 按时间分段：暖场(2min) → 行为回顾(8min) → 核心议题(15min) → 补充探索(5min)
2. 每个核心问题配1-2个追问探针（追问用，不是直接问）
3. 不预设答案方向，保持开放式
4. 标注每道题想获取的信息类型（行为/态度/动机/痛点）
```

---

## 2.4 访谈提纲审查（发给审查模型找问题）

```
请审查以下访谈提纲：

{粘贴提纲全文}

审查标准：
1. 问题顺序是否合理（是否从轻松到深入）
2. 追问探针是否能挖到深层动机（而不是停留在表层）
3. 有没有"是非题"（应该改成开放式）
4. 有没有预设答案方向（比如"你是不是觉得XX太贵了"）
5. 暖场问题是否足够轻松（让用户放松）
6. 时间分配是否合理

逐题审查，给出：✅ / ⚠️ / ❌ + 修改建议。
```
