# Agent Loop 实操指南

> 本章教你如何用两个AI模型互相审查，提升用研质量。

---

## 什么是 Agent Loop？

简单说：**你写初稿 → AI-A帮你改 → AI-B帮你找AI-A漏掉的问题 → 你确认**。

```
你的初稿
    ↓
[主模型] Claude/GPT → 改措辞、补逻辑、优化结构
    ↓
[审查模型] DeepSeek/Grok → 找事实错误、遗漏维度、反例
    ↓
你确认修改 → 完成
```

这比一个人反复检查强，因为不同AI模型的"偏见方向"不同，能互相找到对方的盲区。

---

## 配置方式

### 方式一：手动版（0成本，推荐入门）

**不需要任何API或工具，用浏览器开两个AI对话就行。**

**步骤：**

1. **开第一个对话**（主模型）
   - 用 ChatGPT 或 Claude
   - 把你的初稿发给它，用 `prompt-templates/` 里对应阶段的提示词
   - 拿到修改后的版本

2. **开第二个对话**（审查模型）
   - 用 DeepSeek（免费，https://chat.deepseek.com）
   - 把主模型的输出发给它，用同阶段的"审查"提示词
   - 拿到审查意见

3. **回到第一个对话**
   - 把审查意见发给主模型，让它根据意见修改
   - 如果审查意见有误（你判断的），跳过那条

4. **人工确认**
   - 你做最终判断：哪些修改接受，哪些不接受
   - 不是所有AI意见都对，你的专业判断是最终关口

### 方式二：Hermes Agent自动版（推荐进阶）

如果你用 Hermes Agent，可以把SOP作为skill加载，Hermes会自动执行Agent Loop：

```bash
# 把SKILL.md放到skills目录
cp SKILL.md ~/.hermes/skills/research/user-research-sop/SKILL.md

# 在Hermes对话中说：
"帮我审查这份问卷，用Agent Loop跑一遍"
```

Hermes会自动：
- 用主模型出初稿
- 调用审查模型找问题
- 整合结果给你确认

### 方式三：API版（推荐重度用户）

如果你有多个模型的API Key，可以写脚本自动化：

```python
# 伪代码，展示逻辑
def agent_loop(draft, task):
    # 第1步：主模型优化
    improved = call_model("claude", f"请优化以下{task}：\n{draft}")
    
    # 第2步：审查模型找问题
    review = call_model("deepseek", f"请审查以下{task}：\n{improved}")
    
    # 第3步：主模型根据审查意见修改
    final = call_model("claude", f"根据以下审查意见修改：\n审查意见：{review}\n原文：{improved}")
    
    return final
```

## 审查提示词模板（通用）

不管哪个阶段，审查提示词的基本结构都一样：

```
请审查以下{内容类型}：

{粘贴待审查内容}

审查标准：
1. {标准1}
2. {标准2}
3. ...

请逐项审查，每项给出：
- ✅ 无问题
- ⚠️ 建议修改（附具体建议）
- ❌ 必须修改（附原因和修改方案）

最后给出总体评价：通过 / 需修改 / 需重写
```

## 审查模型怎么选？

| 场景 | 推荐主模型 | 推荐审查模型 | 理由 |
|------|-----------|-------------|------|
| 问卷措辞审查 | Claude | DeepSeek | Claude语感好，DeepSeek逻辑强 |
| 数据分析结论 | DeepSeek | Claude | DeepSeek算数准，Claude能发现过度推断 |
| 报告结构审查 | GPT-4o | Claude | GPT框架感强，Claude细节敏感 |
| 事实核查 | 任意 | DeepSeek+Tavily | DeepSeek推理+外部数据源交叉验证 |

---

## 没有第二个模型怎么办？

**退化方案（效果递减）：**

1. **同一个模型，开两个对话** → 让它分别扮演"作者"和"审查者"
2. **同一个对话，分两轮** → 第一轮出初稿，第二轮说"请以审查者身份找问题"
3. **不用AI，用自查清单** → `templates/Agent-Loop-审查清单.md` 里的检查项，自己逐项过

> 💡 方案3其实是最稳的——清单是你自己经验的沉淀，不依赖AI质量。

## 常见踩坑

| 踩坑 | 后果 | 正确做法 |
|------|------|---------|
| 审查模型说"整体不错" | 没有实质审查 | 提示词里必须要求"逐项审查，给出✅⚠️❌" |
| 审查意见全部接受 | 越改越差 | 你的判断是最终关口，不合理的审查意见要跳过 |
| 无限循环审查 | 永远改不完 | 最多3轮，3轮后仍有分歧提交人工决策 |
| 只用一个模型 | 审查变自我肯定 | 至少开两个不同平台的对话（比如ChatGPT+DeepSeek） |
