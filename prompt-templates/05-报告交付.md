# 阶段五：报告交付 — 提示词模板

---

## 5.1 报告结构生成（发给AI帮你搭框架）

```
请帮我搭建一份研究报告的结构：

研究主题：{什么研究}
研究方法：{问卷/访谈/混合/桌面研究}
核心发现：{列出3-5条最重要的发现}
报告受众：{内部管理层 / 外部合作方 / 产品运营执行层}
报告形态：{PPT / Word报告 / 结论页}

要求：
1. 结构必须是：核心结论前置 → 分模块展开 → 综合建议收底
2. 每个模块必须有三层：洞察提炼 → 结论判断 → 行动建议
3. 管理层报告：第一页就是结论，不铺背景
4. 外部报告：措辞留余地，不用绝对判断
5. 先出结构大纲（标题+每页核心内容一句话），我确认后再填内容
```

---

## 5.2 结论先行改写（发给AI帮你改措辞）

```
请帮我把以下报告段落改成"结论先行"格式：

{粘贴原来的段落}

改写规则：
1. 第一句话必须是结论/判断（不是背景）
2. 数据作为证据跟在结论后面
3. 每段最后一句是行动建议
4. 删除描述性堆砌（"经过分析发现"这类废话）
5. 语气正式克制，不用感叹号
```

---

## 5.3 行动建议生成（发给AI帮你落地）

```
基于以下研究结论，请帮我生成可执行的行动建议：

核心结论：
{列出3-5条结论}

业务背景：
{当前业务情况、可用资源、限制条件}

要求：
1. 每条结论对应1-2条行动建议
2. 每条建议必须包含：做什么 + 谁来做 + 大概什么时候做 + 预期效果
3. 建议必须是可执行的（不能是"加强XX"这种空话）
4. 按优先级排序（投入产出比最高的排前面）
5. 标注每条建议的风险和依赖条件
```

---

## 5.4 报告终审（发给审查模型做最后检查）

```
请对以下报告做终审：

{粘贴报告全文}

审查清单：
1. 主结论是否在第一页/第一屏可见
2. 每条结论是否有数据来源标注
3. 关键数字是否和原始数据一致（如果有原始数据的话）
4. 每条建议是否可执行（能不能直接分配给某人做）
5. 是否有敏感措辞需要处理（给外部看的话）
6. 整体结构是否是"结论→分模块→建议"
7. 是否可以直接交给领导，不需要二次加工

逐项打分：✅ / ⚠️ / ❌，最后给出总体评价：可交付 / 需修改 / 需重写
```
