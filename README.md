# 用户研究标准作业流程（SOP）V2.0

> 基于 Agent Loop 多模型交叉验证的 AI 增强用研流程

覆盖问卷调研、用户访谈、行为数据分析、竞品研究四类场景的标准化流程。

## 核心升级：Agent Loop 质量控制

V2.0 在原六阶段线性流程基础上，引入多模型交叉验证机制：

```
需求承接 → 研究设计 → 数据采集 → 数据分析 → 报告交付 → 收尾
   ↑ [L1]     ↑ [L2]                ↑ [L3]       ↑ [L4]
   └── Loop ──┘─────────────────────┘─────────────┘
```

每个 Loop 节点自动触发：
1. 主模型产出初稿
2. 第二模型审查（事实错误/逻辑漏洞/遗漏维度）
3. 外部数据源交叉验证
4. 人工确认 → 进入下一阶段

## 流程概览

| 阶段 | 核心内容 | Loop节点 |
|------|---------|---------|
| 1. 需求承接 | 需求确认三要素 + 方法选择判断树 | L1 |
| 2. 研究设计 | 样本量规则、问卷规范、访谈提纲规范 | L2 |
| 3. 数据采集 | 问卷投放、访谈执行、行为数据获取 | — |
| 4. 数据分析 | 定量5步 + 定性4步，含质控要点 | L3 |
| 5. 报告交付 | 结构规范 + 交付前自查清单 | L4 |
| 6. 项目收尾 | 进度管理、归档沉淀、方法论复用 | — |
